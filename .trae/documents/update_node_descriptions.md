## 进入场景说明的定义

进入场景说明需要回答三个问题：
1. **什么条件下**会进入这个节点？（前置状态/触发条件）
2. **用户需要做什么**？（在这个节点里要达成的目标）
3. **产出什么**？（离开这个节点时用户能得到什么）

## 实施步骤

### 步骤1：分析每个节点的进入条件
读取各节点文件中的「加载指引」「前序节点」「路由分发表」等章节，提取完整的进入条件。

### 步骤2：统一更新 description
将每个节点的 `description` 改为进入场景说明，格式统一为：
> "当 [前置条件] 时进入。在此节点 [用户要做什么]，产出 [交付物]。"

### 步骤3：验证一致性
检查所有节点的 description 风格是否统一，确保：
- 明确前置条件（某个节点确认、用户意图判断等）
- 说明用户在这个节点需要做什么
- 说明产出什么交付物
- 简短精炼，不展开技术实现细节
