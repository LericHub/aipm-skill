# 知识卡片索引
## LLM使用规则（强制）
当你处理任何产品方案设计相关任务时，必须首先遍历本索引中的所有知识卡片分类，匹配与当前任务主题相关的卡片，检索对应内容作为方案设计的核心参考依据，禁止跳过本步骤直接生成方案。
匹配关键词示例：权限设计、认证体系、用户角色、流程审批、需求评审、产品规划等。

## 三、使用规则（全局唯一）
1. 每个节点执行前，必须读取本索引文件获取卡片配置
2. 所有卡片文件均存储于 `knowledge_cards_path` 目录下，通过相对路径访问
3. LLM执行节点时，必须根据「各节点加载规则」和用户输入匹配加载相关卡片
4. 知识卡片内容作为系统提示的硬约束，必须遵循卡片中的规范输出结果

## 四、加载策略（全局唯一）
知识卡片采用按需加载机制，加载流程如下：
1. **关键词匹配**：提取用户输入中的关键词，与卡片的「激活场景关键词」匹配
2. **节点过滤**：匹配到的卡片需要与当前节点在「适用节点」范围内
3. **内容加载**：读取符合条件的卡片完整内容，作为系统提示的一部分
4. **结果约束**：所有输出必须符合已加载卡片中的规范要求

## 六、API调用规范
所有知识卡片相关操作必须通过以下API实现，禁止直接读取文件：
### 1. load_relevant_knowledge_cards(user_input, current_node)
根据用户输入和当前节点，智能加载相关的知识卡片内容
- 输入参数：`user_input`（用户需求文本）、`current_node`（当前节点名称）
- 返回值：匹配到的所有知识卡片内容列表
- 实现逻辑：严格按照本索引中的「加载策略」执行匹配

### 2. get_knowledge_card(card_name)
获取指定名称的知识卡片完整内容
- 输入参数：`card_name`（卡片中文名称，与卡片目录中的「知识卡片名称」完全匹配）
- 返回值：对应卡片的完整markdown内容
