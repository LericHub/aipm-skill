# 09 - Change (变更分析)

> **注意**：知识卡片加载已由 Router 统一处理。本节点接收 Router 传递的知识卡片内容作为输入。

## 步骤1 职责
- 评估变更影响，制定回退策略
- 变更类型判断
- PRD章节定位
- Penpot画布同步更新
- 版本复制与递增
- 增量变更执行

---

## 步骤3 变更影响分析

### 3.1 变更类型判断

根据用户提交的变更内容，分析变更类型并判断需要重新分析的节点：

| 变更类型 | 判断依据 | 回退节点 |
|---------|---------|---------|
| **想法/新方向** | 用户提交的是一个想法 | brainstorm |
| **功能（涉及业务流程调整或需提交文档）** | 用户提交的是功能，涉业务流程调整 | clarify |
| **功能（涉及新页面和业务）** | 用户提交功能，涉新页面和流程调整 | analysis |
| **文案/布局调整** | 仅涉及文案调整或页面布局 | detail |

### 3.2 PRD章节定位规则

**按大标题定位节点产出**：根据变更涉及的节点，在对应大标题下更新产出内容

| PRD大标题 | 负责节点 | 说明 | Penpot同步 |
|-----------|---------|------|-----------|
| 项目概述 | brainstorm/clarify | 需求背景和目标 | - |
| 需求分析 | analysis | 业务流程和页面列表 | 用户故事页面 |
| 详细设计 | detail | 事件列表和异常场景 | **PRD 详细设计页面** |
| 页面详细设计 | detail | 按页面维度组织详细内容 | **PRD 详细设计页面** |
| 原型与交互 | genPrototype/genHTML | 原型引用和交互说明 | 原型设计页面 |
| 变更记录 | change | 变更历史记录 | - |

**章节存在性检查：**
- 更新PRD前，先检查目标章节是否存在于当前PRD文档中
- **如果章节不存在**：说明该节点尚未执行或内容未生成，此时：
  - 不创建空章节占位
  - 在变更影响分析报告中标注"该章节尚未生成，将在对应节点执行时自动创建"
  - 仅更新已存在的章节内容
- **如果章节存在**：按正常流程更新该章节内容

**按页面名称定位具体内容**：在节点产出内按页面名称查找需要更新的页面和事件

### 3.3 Penpot画布同步

变更涉及Penpot画布时，执行以下同步策略：

| 变更场景 | 同步方式 | 操作 |
|---------|---------|------|
| 用户故事变更 | 按名称定位Board | 替换Board内的文本元素 |
| 页面设计变更 | 按名称定位Board | 清除旧设计稿后重新绘制 |
| 仅文案修改 | 按名称定位Text | 修改 `text.characters` 属性 |
| **PRD 内容变更** | **按栏标题定位 Board** | **替换栏内的子容器内容** |
| **新增 PRD 章节** | **在根容器中追加新栏 Board** | **使用 Flex row 自动排列** |
| **删除 PRD 章节** | **按名称定位栏 Board 并移除** | **后续栏自动左移** |
| **表格内容变更** | **按表格名称定位，替换数据行** | **保持表头不变** |
| **流程图变更** | **按流程图名称定位，重新绘制节点** | **清除旧节点，绘制新节点** |

**避免重叠策略**：
- 使用Flex布局自动排列，修改时只替换内部元素
- 清除旧内容时从后往前遍历删除：`for (var i = kids.length-1; i >= 0; i--) kids[i].remove()`
- 保持容器结构不变，避免坐标重算
- **PRD 页面更新时保持栏容器结构，只替换内部子容器**
- **新增栏时直接 appendChild 到根容器，Flex 自动排列**
- **删除栏时调用 `board.remove()`，Flex 自动收缩**
- **表格更新时只替换数据行，保持表头和列宽**

### 3.4 版本号管理规则

- **涉及前序内容修改**：major+1（V1.x → V2.0）
- **仅为新增功能**：minor+1（V1.0 → V1.1）

## 步骤4 变更执行流程

### 4.1 接收用户变更内容

1. 接收用户描述的变更内容
2. 分析变更属于哪种类型（想法/功能/文案布局）
3. 判断需要回退的节点

### 4.2 变更影响分析

1. 读取当前版本PRD文档
2. 按大标题定位需要更新的章节
3. 按页面名称定位需要更新的具体内容
4. 判断是否需要同步Penpot画布
5. 展示变更影响分析报告

```
## 变更影响分析报告

### 变更类型
- 类型：[想法/功能/文案布局]
- 说明：[变更内容的简要描述]

### 影响章节
- [ ] PRD文档章节1
- [ ] PRD文档章节2

### Penpot同步
- [ ] 用户故事页面：需要/不需要
- [ ] **PRD 详细设计页面：需要/不需要**
- [ ] 原型设计页面：需要/不需要

### 建议回退节点
- 回退到：[节点名称]
- 理由：[判断依据]

### 版本策略
- 版本递增：[major/minor]
- 新版本号：[Vx.x → Vx.x]
```

### 4.3 版本复制

确认变更影响分析后，执行版本复制：

1. 复制当前PRD文档为新版本文件：`PRD_{Version}.md`
2. 复制当前prototype目录为新版本目录
3. 在新版本PRD文档中标记变更记录
4. 如需同步Penpot：在现有页面上更新内容（Penpot不支持版本复制，仅更新）

### 4.4 回退执行

1. 调用 manage.transition_to_first_affected_node(受影响节点列表)
2. 更新Memory.json的版本号和文件路径
3. 回退到第一个受影响的节点重新执行

---

## 步骤5 完成后

1. 展示变更影响分析报告
2. 等待用户确认
3. 确认后 → 执行版本复制（PRD文档和prototype目录）
4. 确认后 → 同步Penpot画布（如需要）
5. 确认后 → 更新Memory.json的版本号和文件路径
6. 确认后 → 调用 manage.update_node_status("change", "CONFIRM")
7. 确认后 → 调用 manage.transition_to_first_affected_node(受影响节点列表)
8. 确认后 → 调用 manage.persist_memory()
9. 回退到第一个受影响的节点
