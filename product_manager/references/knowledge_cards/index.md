# 知识卡片索引
## LLM使用规则（强制）
当你处理任何产品方案设计相关任务时，必须先由 Router 读取本索引，再结合当前节点和用户输入决定要使用哪些知识卡片。业务节点直接消费 Router 提供的知识卡片上下文，禁止在业务节点中重新定义独立加载策略。
匹配关键词示例：权限设计、认证体系、用户角色、流程审批、需求评审、产品规划等。

## 三、使用规则（全局唯一）
1. Router 作为唯一入口，在每次对话开始时必须先读取本索引，再统一选择并加载知识卡片，传递给目标节点
2. 业务节点不再重复执行知识卡片加载步骤，直接接收 Router 传递的卡片内容
3. 所有卡片文件均存储于 `knowledge_cards_path` 目录下，通过相对路径访问
4. Router 选择卡片时，必须结合用户输入、当前节点和本索引中的卡片目录/适用节点/使用规则进行判断
5. 知识卡片内容作为系统提示的硬约束，必须遵循卡片中的规范输出结果

## 四、加载策略（全局唯一）
知识卡片采用按需加载机制，加载流程如下：
1. **Router读取索引**：每轮开始先读取本索引，获取卡片目录和规则
2. **关键词匹配**：提取用户输入中的关键词，与卡片的「激活场景关键词」匹配
3. **节点过滤**：匹配到的卡片需要与当前节点在「适用节点」范围内
4. **LLM自主选择**：由 Router 基于上下文决定需要加载哪些卡片
5. **内容加载**：读取符合条件的卡片完整内容，作为系统提示的一部分
6. **结果约束**：所有输出必须符合已加载卡片中的规范要求

## 五、Router选择提示（全局唯一）
Router 读取本索引后，可参考以下倾向进行选择，但不将其视为硬编码默认表：

| 目标节点 | 优先关注的卡片方向 | 选择提示 |
|------|----------|------|
| BRAINSTORM | 需求分析方法论、会员业务中台、埋点、权限、活动配置、页面装修 | 优先帮助用户发散和搭建功能骨架 |
| CLARIFY | 会员中台、系统边界、业务规则 | 优先帮助用户澄清边界和规则，避免技术实现提问 |
| ANALYSIS | 方法论、会员规范、权限、活动配置、页面装修 | 优先支撑 PRD 结构化分析 |
| DETAIL | 方法论、会员规范、埋点、权限、页面装修 | 优先支撑页面级详细设计和事件定义 |
| DRAWUSERSTORY / GENPROTOTYPE / GENHTML / CHANGE | 按需选择 | 根据当前主题和输出目标动态挑选 |

## 六、API调用规范
所有知识卡片相关操作由 Router 发起，业务节点不直接定义 API 约定。

### 1. Router读取索引
- 输入：`user_input`、`target_node`
- 动作：读取本索引，分析适用卡片并决定是否加载

### 2. Router读取卡片内容
- 输入：`card_name`
- 动作：按相对路径读取对应卡片完整内容并传递给目标节点
