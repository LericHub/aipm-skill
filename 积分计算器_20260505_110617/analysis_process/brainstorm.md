# 功能策划

## 背景与痛点

> 用户原始回答：搭建一个积分计算器应用，天猫小程序，从0-1搭建

**归纳扩展**：
目前在会员运营场景中，用户购买商品前无法直观知道能获得多少积分，导致购买决策缺乏积分激励的透明度。需要一个面向 C 端会员的小程序工具，让用户能选商品、算积分，提升积分兑换的参与度和购买转化。

## 业务目标

> 用户原始回答：帮助用户计算购买商品可获得的积分

**归纳扩展**：
通过透明化的积分试算，让用户"所见即所得"地知道买哪些商品能获得多少积分，从而激励用户完成购买行为。B 端管理端则支撑活动运营，配置推荐商品、监控活动效果。

## 功能骨架

1. 会员小程序端（C端）
   - 会员信息展示
     - 当前积分余额
     - 当前会员等级
   - 商品推荐列表
     - 展示后台配置的推荐商品
     - 点击商品跳转淘宝商品详情页
     - 支持多选商品
   - 积分试算
     - 用户选中多个商品后点击"计算"按钮
     - 调用会员中台积分试算接口（DL 2.2.95）
     - 展示预计可获得积分数量

2. 管理端（B端）
   - 多店铺兑礼活动管理
     - 创建活动
     - 启用/禁用活动
     - 活动配置编辑（活动时间、推荐商品）
   - 商品配置
     - 从淘宝批量获取商品信息（taobao.items.seller.list.get）
     - 配置推荐商品列表
   - 信息看板
     - 活动曝光埋点
     - 用户行为日志（关注C端异常操作）
