# PRD V1.0 - 积分计算器应用

> 文档版本：V1.0
> 更新日期：2026-05-03
> 文档状态：初版

---

## 一、项目概述

### 1.1 项目背景
搭建一个天猫小程序积分计算器应用，帮助会员计算购买推荐商品可获得的积分数量，提升会员对积分价值的感知，促进会员活跃度。

### 1.2 项目范围
- **项目类型**：天猫小程序（从0-1搭建）
- **目标端口**：会员小程序端（C端）+ B端管理端
- **核心目标**：会员积分试算工具 + 多店铺兑礼活动管理平台

### 1.3 术语定义
| 术语 | 说明 |
|------|------|
| 积分试算 | 根据商品价格和会员信息，预估购买后可获得的积分数量 |
| 兑礼活动 | B端配置的积分相关营销活动，包含推荐商品和活动时间 |
| 会员中台 | 会员信息中台系统，提供积分、等级查询及试算接口 |

## 二、需求分析

### 2.1 用户画像

#### C端用户 - 天猫会员
- **特征**：已注册天猫会员，关注积分价值
- **诉求**：快速了解购买商品能获得多少积分
- **场景**：浏览推荐商品 → 选择感兴趣的商品 → 计算积分 → 决策是否购买

#### B端用户 - 店铺运营
- **特征**：负责店铺积分营销活动配置
- **诉求**：便捷创建和管理兑礼活动，监控活动效果
- **场景**：创建活动 → 配置商品 → 启用活动 → 查看数据

### 2.2 核心业务流程

```
【C端核心流程】
用户进入小程序
    ↓
自动查询会员信息（积分+等级）
    ↓
展示当前进行中的兑礼活动及推荐商品
    ↓
用户选择商品（可多选，不限数量）
    ↓
点击"计算积分"按钮
    ↓
调用积分试算接口（传入商品价格）
    ↓
展示计算结果（总积分+明细）
```

```
【B端核心流程】
运营人员登录管理端
    ↓
创建兑礼活动（名称、时间、规则）
    ↓
配置推荐商品（从淘宝同步/手动添加）
    ↓
保存并启用活动
    ↓
监控活动数据（曝光、用户行为）
```

### 2.3 功能模块

#### C端小程序功能

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 会员信息展示 | 展示会员当前积分、等级 | P0 |
| 活动商品列表 | 展示当前活动推荐商品（图片、名称、价格） | P0 |
| 商品选择 | 多选商品，不限数量 | P0 |
| 积分计算 | 调用试算接口，展示结果 | P0 |
| 结果展示 | 总积分、各商品积分明细 | P0 |

#### B端管理端功能

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 活动管理 | 创建、编辑、启用/禁用活动 | P0 |
| 商品配置 | 同步淘宝商品、配置推荐商品 | P0 |
| 活动配置 | 时间配置、商品关联 | P0 |
| 信息看板 | 活动曝光PV/UV、用户行为日志 | P1 |

## 三、功能需求

### 3.1 C端小程序

#### 3.1.1 会员信息展示
- **触发时机**：进入小程序首页自动查询
- **查询接口**：DL-3.2.7（会员当前积分）、DL-3.2.4（会员等级）
- **展示内容**：
  - 会员等级标识
  - 当前可用积分
- **异常处理**：
  - 查询会员失败：显示 Toast 提示
  - 非会员用户：引导入会流程，入会失败显示 Toast 提示
- **刷新机制**：每次回到首页自动重新查询

#### 3.1.2 活动与商品展示
- **展示规则**：展示当前进行中的兑礼活动及其推荐商品
- **商品信息**：商品图片、商品名称、商品价格
- **空状态**：无进行中的活动时展示空状态提示

#### 3.1.3 商品选择
- **交互方式**：点击商品卡片进行选择/取消选择
- **选择标识**：已选商品显示选中状态
- **数量限制**：不限数量
- **已选汇总**：底部展示已选商品数量和总价

#### 3.1.4 积分计算
- **触发方式**：点击"计算积分"按钮
- **入参**：已选商品的价格列表
- **调用接口**：DL 2.2.95 积分试算接口
- **防重机制**：计算中禁用按钮，防止重复点击
- **异常处理**：接口失败时显示友好提示

#### 3.1.5 结果展示
- **展示内容**：
  - 预估获得总积分
  - 各商品积分明细（商品名称 + 对应积分）
- **关闭方式**：点击关闭或蒙层返回商品列表

### 3.2 B端管理端

#### 3.2.1 兑礼活动管理
- **活动列表**：
  - 展示字段：活动名称、活动时间、状态、操作
  - 筛选：按状态、时间范围筛选
  - 分页：支持分页查询
- **创建活动**：
  - 活动名称（必填）
  - 活动时间（开始时间、结束时间）
  - 活动规则说明（选填）
- **编辑活动**：
  - 仅禁用状态的活动可编辑
  - 编辑后保存为草稿或启用
- **启用/禁用**：
  - 禁用的活动可启用
  - 进行中的活动可禁用（立即失效）

#### 3.2.2 商品配置
- **商品来源**：
  - 从淘宝同步商品信息（taobao.items.seller.list.get）
  - 输入淘宝商品ID批量获取
- **商品字段**：商品ID、商品名称、商品价格、商品图片
- **推荐配置**：将商品关联到活动，设置展示顺序
- **商品复用**：同一商品可关联到多个活动

#### 3.2.3 信息看板
- **活动曝光数据**：
  - PV（不去重）：每次进入小程序且活动进行中计1次
  - UV（去重）：同一用户多次进入只计1次
- **用户行为日志**：
  - 记录接口调用异常
  - 记录页面加载异常
  - 记录计算失败异常
- **数据展示**：按活动维度汇总，支持时间筛选

### 3.3 系统对接需求

| 对接系统 | 接口 | 用途 | 优先级 |
|----------|------|------|--------|
| 会员信息中台 | DL-3.2.7 | 查询会员当前积分 | P0 |
| 会员信息中台 | DL-3.2.4 | 查询会员等级 | P0 |
| 会员信息中台 | DL 2.2.95 | 积分试算 | P0 |
| 淘宝开放平台 | taobao.items.seller.list.get | 批量获取商品信息 | P0 |

## 四、非功能需求

### 4.1 性能需求
- 页面首屏加载时间 < 2s
- 积分试算接口响应时间 < 1s
- 商品列表支持懒加载

### 4.2 可用性需求
- 小程序兼容主流手机机型
- 管理端兼容 Chrome、Safari 等主流浏览器

### 4.3 安全需求
- B端管理端需要登录认证
- 数据权限隔离：用户只能看到授权店铺的数据
- 接口调用需要鉴权

### 4.4 埋点需求
- 活动曝光：PV、UV统计
- 用户行为：接口异常、页面异常、计算异常
- 不关注订单成交转化

## 六、状态定义

### 6.1 活动状态

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 草稿 | 创建后未启用 | 编辑、启用、删除 |
| 未开始 | 已启用，未到开始时间 | 禁用、编辑 |
| 进行中 | 在活动时间范围内 | 禁用 |
| 已结束 | 超过结束时间 | 仅查看 |
| 已禁用 | 手动禁用 | 启用、编辑 |

### 6.2 商品状态

| 状态 | 说明 |
|------|------|
| 已同步 | 从淘宝同步成功 |
| 已关联 | 已关联到活动 |
| 已下架 | 商品失效或手动下架 |

## 七、异常处理

### 7.1 C端异常

| 异常场景 | 处理方式 |
|----------|----------|
| 查询会员失败 | Toast 提示"会员信息获取失败，请稍后重试" |
| 非会员用户 | 引导入会，入会失败 Toast 提示 |
| 积分试算失败 | Toast 提示"计算失败，请稍后重试" |
| 无进行中的活动 | 展示空状态页面 |
| 网络异常 | 全局网络异常提示 |

### 7.2 B端异常

| 异常场景 | 处理方式 |
|----------|----------|
| 淘宝商品同步失败 | 提示具体失败原因，支持重试 |
| 活动时间冲突 | 表单校验提示 |
| 无权限访问 | 跳转无权限页面 |
