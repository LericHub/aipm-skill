# 活动配置分层范式

## 1. 当时为什么这么决策

**背景**：当前已经建设多个营销互动小程序，公用一个后台，用户从后台菜单栏进入不同的应用
 - 当前已经形成一个应用后台配置规范，延续同样的信息架构，能够有效降低用户的学习成本
 - 基础配置：所有活动都要有，包括活动名称、时间、说明、小程序分享封面等通用信息
 - 活动配置：不同活动类型有差异的参数，需要按照实际业务场景设计配置表单，例如权益卡券活动、抽奖活动、会员权益活动
 - 活动装修：每个活动需要上传的素材各异，设计师会按照活动的实际情况设计素材，

**边界条件**：如果活动之间差异极小（只有时间不同），可能不需要"活动装修"层；如果活动种类极其单一，可能只需要"基础配置"。

---

## 2. 通常落地方案是什么
以CRM权益小程序为例：
**背景**
- 通过小程序为会员发放卡券权益，小程序可展示品牌给会员发放的全渠道权益卡券，用户从小程序上领取卡券并获得相应平台的权益（积分、优惠券等）
- 需要在活动中配置活动需要发放的卡券，并配置奖品参数（平台卡券id、会员中台行为积分编码等），从UX角度出发单独将这部分的配置项抽取出来，作为活动配置层的子项
- 小程序以活动维度运营，不同的活动发放的卡券各异，奖品各异。需要在活动层面配置每一档活动的卡券卡面图片
 
**设计结构**：
```
活动配置
├── 基础配置（所有活动必须有）
│   ├── 活动名称、活动说明
│   ├── 开始时间、结束时间
│   ├── 领取开始时间、领取结束时间：属于CRM权益小程序特别的配置字段；用于活动预热期展示活动卡券，但是不允许领取
│   └── 小程序分享封面
├── 活动配置
│   ├── 活动类型：生日礼/升级礼/通用活动，不同活动类型的业务流程不同
│   ├── 活动规则：富文本输入框输入活动规则，用于前端展示
│   └── 活动权益：输入要发放的中台权益卡券编号和奖品参数（平台卡券id、会员中台行为积分编码等）
└── 活动装修
    ├── 卡券封面图：因为想要在卡券封面上显示权益内容（积分、优惠券等）和状态（未卡券、已使用等），所以需要针对权益+状态进行配置
    └──服务预约素材：只针对服务预约活动，配置展示服务项目的说明介绍图片
```

## 3. 类似场景怎么考虑
**优先考虑**：考虑复用现有的配置框架，延续统一的信息架构提高用户的使用效率

**适用检查点**：
- 当前项目是否有一些业务层面的差异，需要设置应用专用的配置字段
- 结合页面考虑哪些装修素材是随不同活动而变化的，需要在活动层面设计配置项，例如卡券卡面
- 活动配置参数是否复杂到需要分成三层配置？
    - 若活动之需要简单配置参数，可以只保留基础配置层
    - 通常涉及到活动层面的装修，才需要设计活动装修层

**常见变体**：
- 中等复杂度：基础 + 活动配置
- 复杂活动：三层全有
