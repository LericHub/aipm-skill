## 步骤5 技能描述步骤一：需求分析

**🚨 关键规则：初步的 PRD 也必须非常详细，不能只是一个空框架。**

### 1.1 产出内容

根据步骤一的确认信息，直接生成一份详细的初步 **Markdown 格式**文档（同时将文档内容以文本方式输出给用户）。这份初版 PRD 必须搭建好完整的文档结构骨架，并且必须深度填充以下部分：

1. **项目基本信息**
2. **需求背景与目标**：先说明需求背景，然后同列表的方式展示业务目标。业务目标必须包含四列表格（目标类型、描述、衡量指标、目标值）。
3. **识别用户类型和端口**：
    - 识别用户类型和端口。
    - 根据用户类型和端口，梳理用户在使用产品时的场景和核心流程。（先使用文字描述流程节点，如用户下单-》订单确认-》订单支付）
4. **识别业务对象**：用 Markdown 表格输出业务对象和关键属性（如用户、订单、商品等），表格格式如下：

```markdown
| 业务对象 | 对象属性 |
|---------|---------|
| 用户 | 用户ID、用户名、手机号、邮箱、注册时间、最后登录时间、用户等级、积分余额 |
| 订单 | 订单ID、用户ID、订单状态、订单金额、支付方式、下单时间、支付时间 |
| 商品 | 商品ID、商品名称、商品描述、商品价格、库存数量、兑换所需积分、上架状态 |
```

**🚨 必须使用上述表格格式！**

5.  **业务流程图**：
  - 用 Mermaid 图表输出业务流程图，展示用户在使用产品时的场景和核心流程。
  - 必须使用横向的流程图

6. **拆解用户故事**：将业务需求拆解成多个用户故事
```markdown
| 故事编号 | 端口 | 用户故事 |
|---------|---------|---------|
| S01 | 小程序端 | 作为一个消费者，我需要浏览商品列表、查看商品详情、下单、支付、查看订单详情 |
| S02 | 管理端 | 作为一个管理员，我需要配置活动的参数 |
```
7. **页面列表**：根据用户故事整理页面列表
  - 用列表输出B、C端页面列表，需要按照页面个能和用户操作两个维度描述。
8. **埋点方案**：必须使用表格输出埋点场景的埋点的字段。
9. **异常场景**：对接系统返回的异常状态处理。（接口调用异常、接口返回业务异常）
*（后续章节如详细设计、页面详细设计、原型与交互等，在对应节点完成后自动追加到PRD中，当前analysis节点不生成这些章节）*

### 1.2 用户确认 【等待用户确认】

询问用户：这是产品的详细第一版架构和业务逻辑，您看方向和基础逻辑准确吗？如果没问题，我们将先进行原型设计，通过具体的画面来进一步理清交互细节和可能遗漏的功能。

**🛑 强制暂停：在此处必须停止输出！等待用户回复同意后，才能进入步骤四。**

## 步骤6 输出文件

### 6.1 目录初始化

在analysis节点完成时，创建版本目录并生成PRD文档框架：

```
./{项目名称}_{datetime}/output/V1/
├── PRD_V1.md              # PRD文档（初始版本）
└── prototype/             # 原型目录（在genHTML节点创建）
```

### 6.2 PRD文档框架模板

生成PRD文档，框架包含以下章节。

**🚨 关键规则：只包含当前已生成内容的章节，未生成内容的章节不出现在PRD中。**

- analysis节点首次生成时，只包含第1、2章节
- detail节点完成后，追加第3、4章节
- genHTML节点完成后，追加第5章节
- change节点维护第6章节

**analysis节点首次生成的PRD框架：**

```markdown
# PRD文档 - V1.0

## 1. 项目概述
### 1.1 需求背景与目标 [brainstorm/clarify节点产出]
### 1.2 用户角色

## 2. 需求分析
### 2.1 业务流程图 [analysis节点产出]
### 2.2 页面列表
### 2.3 功能列表
```

**完整PRD框架（所有节点完成后）：**

```markdown

### 1.1 需求背景与目标 [brainstorm/clarify节点产出]

### 3.1 页面流程图 [detail节点产出]

## 4. 页面详细设计

#### 4.1.2 功能描述

## 5. 原型与交互
```

### 6.3 输出路径

- 文件名: `PRD_V1.md`
- 路径: `./{项目名称}_{datetime}/output/V1/PRD_V1.md`
- 生成规则: 见 [01-router.md](01-router.md) 中的文件生成规则

## 步骤7 完成后

1. 创建版本目录：`./{项目名称}_{datetime}/output/V1/`
2. 读取analysis_process/clarify.md内容（brainstorm和clarify整合内容）
3. 生成PRD文档框架，将brainstorm/clarify内容填入"项目概述"章节
4. 将analysis节点产出填入"需求分析"章节
5. 保存产出到文件：`./{项目名称}_{datetime}/output/V1/PRD_V1.md`
6. 调用 manage.update_node_output("analysis", "{项目名称}_{datetime}/output/V1/PRD_V1.md")
7. 展示产出摘要
8. 等待用户确认
9. 确认后 → 调用 manage.update_node_status("analysis", "CONFIRM")
10. 确认后 → 调用 manage.transition_to_next_node()
11. 确认后 → 调用 manage.persist_memory()
12. 进入 DrawUserStory 节点
